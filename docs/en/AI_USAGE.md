# AI Usage Policy

## Overview

This project uses AI-assisted tools (notably ChatGPT)
as part of the development workflow.

The goal is to accelerate reasoning, exploration, and validation
— **not to automate code production**.

---

## Core Principles

### 1. Human Ownership

All code in this repository is:

* reviewed
* validated
* understood

by a human before being written or committed.

There is **no blind copy-paste or automated code generation pipeline**.

---

### 2. No Autonomous Generation

AI tools are used for:

* reasoning assistance
* architecture discussions
* alternative approaches exploration

AI tools are **not used to directly generate production-ready code without review**.

---

### 3. Systematic Review & Challenge

Every AI-generated suggestion is:

* critically reviewed
* challenged when necessary
* rejected if ambiguous, unnecessary, or misaligned with project constraints

---

### 4. Final Responsibility

All implementation decisions are made by the project author.

Regardless of the source of an idea (AI or otherwise),
**the final code is always written or validated manually**.

---

## Security Considerations

* No secrets or sensitive data are shared with AI tools
* No external code is trusted without verification
* All dependencies and logic are validated independently

---

## Future Evolution

If AI agents or automated workflows are introduced in the future,
this document will be updated to include:

* agent rules
* scope of autonomy
* validation constraints

---

## Philosophy

AI is treated as a **tool for augmentation**, not delegation.

The integrity, security, and determinism of the core
remain fully controlled and auditable.
