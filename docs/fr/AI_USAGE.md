# Politique d'utilisation de l'IA

## Vue d'ensemble

Ce projet utilise des outils assistés par IA (notamment ChatGPT)
dans le cadre du workflow de développement.

L’objectif est d’accélérer le raisonnement, l’exploration et la validation
— **et non d’automatiser la production de code**.

---

## Principes fondamentaux

### 1. Responsabilité humaine

Tout le code de ce dépôt est :

* relu
* validé
* compris

par un humain avant d’être écrit ou commit.

Il n’existe **aucun copier-coller aveugle ni pipeline de génération automatique de code**.

---

### 2. Aucune génération autonome

Les outils d’IA sont utilisés pour :

* assister le raisonnement
* discuter de l’architecture
* explorer des approches alternatives

Les outils d’IA **ne sont pas utilisés pour générer directement du code prêt pour la production sans relecture**.

---

### 3. Revue et remise en question systématiques

Chaque suggestion générée par l’IA est :

* analysée de manière critique
* remise en question si nécessaire
* rejetée si ambiguë, inutile ou non alignée avec les contraintes du projet

---

### 4. Responsabilité finale

Toutes les décisions d’implémentation sont prises par l’auteur du projet.

Quelle que soit l’origine d’une idée (IA ou autre),
**le code final est toujours écrit ou validé manuellement**.

---

## Considérations de sécurité

* Aucun secret ni donnée sensible n’est partagé avec les outils d’IA
* Aucun code externe n’est utilisé sans vérification
* Toutes les dépendances et la logique sont validées indépendamment

---

## Évolution future

Si des agents IA ou des workflows automatisés sont introduits à l’avenir,
ce document sera mis à jour pour inclure :

* les règles des agents
* le périmètre d’autonomie
* les contraintes de validation

---

## Philosophie

L’IA est considérée comme un **outil d’augmentation**, et non de délégation.

L’intégrité, la sécurité et le déterminisme du core
restent entièrement maîtrisés et auditables.
